# Catsystem3引擎解包工具

**该工具用于处理catsystem2引擎的安卓版catsystem3(好像叫这个?)**  
## 运行方法：
- 命令行窗口方式：运行release的cs3_toolsCLI.exe或在cmd里运行
```javascript
python CLI_start.py
```
并根据里面的提示操作即可
- 交互界面方式：运行release的cs3_toolsGUI.exe或在cmd里运行
```javascript
python GUI_start.py
```
**注意，使用GUI_start.py打开需要先安装PYQT5库**  

- json版提取/回封说明：提取时选择剧情dat文件，然后会在程序目录下解包出iss原版文件和解压后的bin文件，并提取出文本在iss文件同级的json文件夹里。json文件有两个格式，第一种是一开始设计时决定的字典套字典格式，每个语句组的人名和文本都在一个字典内；第二种是后面适配vnt另外写的列表套字典格式；请根据自己需要选择。同时提取时会生成名字字典在name_dict文件夹，名字字典仅提供参考，不参与回封过程。回封时先把改动好的json文件放在剧本目录下cn_json文件夹里(参考路径scene/cn_json/)，再使用程序选择剧本文件夹目录，例如解出来是scene/iss这种目录就选择scene文件夹就可以了，最后再选择原剧情文本存在的dat文件，等待回封完成~~(注：vnt格式回封时会生成同名bak文件在同目录下方便测试和备份)~~(已经优化了)  
  P.S. 据我发现，有些游戏会把文本放在普通封包里，作为文本封包的data_scene_01.dat只是备份，如果你发现改了data_scene_01.dat这种文本封包没反应后，可以通过本工具的提取文件信息表找找是不是哪个普通封包有文本(iss文件)。  

- txt版提取/回封说明：提取时选择剧情dat文件，然后会在程序目录下解包出iss原版文件和解压后的bin、txt文件放在对应的文件夹，同时程序内置了正则功能，会自动把正则提取后的文本放在跟iss文件夹同级的cn_re_txt文件夹里，有需要翻译的可以用这个正则。其次，提取的txt用的是\n\n换行，如果有自己写正则需求的请注意。回封时如果需要回封cn_re_txt文件夹的文本的请选择正则回封模式，程序会先提取cn_re_txt的文本生成汉化后的txt在cn_txt文件夹里再回封成iss。**读取文件夹优先级是cn_re_txt>cn_txt>原版txt文件夹**。如果你有单独更改的txt文件(例如更改了命令)，需要删除cn_re_txt的同名文件或者不使用正则回封功能，然后把txt文件放在cn_txt文件夹里回封。不使用正则回封情况下只读取cn_txt文件夹里的txt文件进行回封。这里提一嘴，可以先用正则文本翻译完，然后生成新txt到cn_txt文件夹后再在文件夹里单独改更新的txt里的命令或文本，最后使用不正则回封再打包一次。需要注意的是请不要随意改动换行、文本里的\n和首尾的双引号，如因此引起的重封包bug不负责。并且改动解出来的原版txt时不要破坏txt里的null(用emeditor这种编辑器会把0x000x00这种null改写成0x200x20，目前推荐使用vscode更改)。  

- dat解封包说明：提取时只需要选择dat封包就会把里面的文件解出来放在程序目录的cs3_extract_file文件夹下。回封时先把需要重封包的文件放在程序目录下的Repack_file文件夹里，再选择你需要重封包的dat文件即可，新封包生成在Packed_dat文件夹里。要注意**重封包前必须先解包这个dat获得所有文件，其次是回封后的包最好不要大于100m甚至150m**，否则可能会出问题。批量提取之后不知道要回封哪几个包的可以提文件信息表查一下，并且回封时有新文件的话会打印新文件的文件名在控制台。  
  ↑ *为此特意保留了GUI的控制台hhh*  
  P.S. (此功能多数用于图片，不过图片用的是hg3格式需要另找工具，反正前人的工具备矣)。  

>另外，对于该引擎安卓版的汉化，还需要改动apk里lib/arm64_v8a/fescript.so文件的toSJISString()函数，不然替换文本后会闪退；至于32位的armeabi_v7a/fescript.so没得救了，主包改了好多个点都不太行，建议直接删掉32位库，反正没多少手机用了。还有就是改下assets文件夹下的字体，不然有些汉字显示不了。

  那端出来之后就有很多手机玩家入甜坑玩了，这个时候问题又来了，甜2+是甜2的后日谈，甜2的安卓没搞定反而后日谈的安卓搞定了，只能用手机的玩家玩后日谈就会玩的一知半解纯看糖了(虽说废萌主打就是吃糖为主)。那时候圈子里关于我们刚发的安卓版热度很高，考虑到大家的游玩体验，哥们修完bug想了一天，决定还是得动手研究一下甜2这套引擎怎么搞。于是五一假修了四五天bug后又去图书馆鏖战了几天，结果打着打着灵感来了，写了个很粗糙的脚本，好歹是把文本文件和文本都提出来了。这个时候组那边担心封不回去就一直拖着没动，我又得回去研究回封。回封的路上总有神神奇奇的小bug，说到底还是对封包信息研究不够。我又只能回去重新研究包的信息……等到一周之后带着我的优化版再度出山。这个时候提取和回封都搞定了，按理说应该很快就能出了。但是由于我提取的文本太乱了(参考工具的提取txt功能)，不好翻译，其次是这个沟槽的引擎有一个校验sjis字符的函数没爆破掉，搞得我换完文本跑两句就闪退，真没辙了，只能一直拖着。虽然说有所进展，但是卡在校验这里一直过不去，有些文本不会闪退，有些又会，完全做不了汉化。记得当时挂着ida研究了很久还是没搞懂，只能暂且放弃了。
